# Getting Started - Step by Step

## First Time Setup

### 1. Install Dependencies

**Install Node packages:**

```bash
npm install
```

**Install Flutter dependencies:**

```bash
cd app
fvm flutter pub get
cd ..
```

### 2. Setup Database

- Create a MySQL database
- Import your schema/tables
- Update `.env` with database credentials

### 3. Create Environment Files

**Root `.env` file:**

```env
# Database
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=gg_store

# API
PORT=3000
JWT_SECRET=your_secret_key_here

# Ngrok (for connecting Flutter to API)
NGROK_API_URL=http://localhost:3000
NGROK_API_TIMEOUT=30000
```

**Flutter `.env` file** (will be auto-generated by sync-env script):

```bash
# This file is at app/.env
# It gets created automatically when you run npm run dev
API_BASE_URL=http://10.0.2.2:3000
API_TIMEOUT=30000
```

## Running the Project

### Option 1: Run Everything Together (Recommended)

1. **Launch Android Emulator** from Android Studio or:

   ```bash
   # List available emulators
   emulator -list-avds

   # Start an emulator
   emulator -avd Pixel_5_API_34
   ```

2. **Open VS Code** and go to Run & Debug panel (Ctrl+Shift+D)

3. **Select "Full Stack (Flutter + API)"** from the dropdown

4. **Press F5** or click the green play button

   This will:
   - Start the Express API with nodemon (hot reload)
   - Start ngrok tunnel (if configured)
   - Sync the API URL to Flutter's .env
   - Launch the Flutter app on the emulator

### Option 2: Run Separately (For Debugging)

**Terminal 1 - Start API:**

```bash
npm run dev
```

Wait until you see:

```
Server running on port 3000
Ngrok tunnel: https://xxxx-xx-xx-xx-xx.ngrok.io
```

**Terminal 2 - Start Flutter:**

```bash
cd app
fvm flutter run
```

Or use VS Code:

- Run & Debug → "Flutter (Mobile App via FVM)" → F5

### Option 3: Production Mode

**Start API only:**

```bash
npm start
```

**Build Flutter APK:**

```bash
cd app
fvm flutter build apk --release
```

## Connecting Flutter to API

### For Android Emulator:

The API runs on `localhost:3000` on your machine, but the emulator sees it as:

- `10.0.2.2:3000` (Android emulator special IP)

Update `app/.env`:

```env
API_BASE_URL=http://10.0.2.2:3000
API_TIMEOUT=30000
```

### For Physical Device (same WiFi):

1. Find your computer's local IP:

   ```bash
   # Windows
   ipconfig
   # Look for IPv4 Address (e.g., 192.168.1.100)

   # Mac/Linux
   ifconfig
   ```

2. Update `app/.env`:

   ```env
   API_BASE_URL=http://192.168.1.100:3000
   API_TIMEOUT=30000
   ```

3. Make sure your firewall allows port 3000

### Using Ngrok (for remote testing):

The `npm run dev` script automatically:

1. Starts ngrok tunnel
2. Syncs the ngrok URL to `app/.env`

Your Flutter app will connect via the public ngrok URL.

## Troubleshooting

### API won't start

```bash
# Check if port 3000 is already in use
# Windows
netstat -ano | findstr :3000

# Kill the process using the port
taskkill /PID <PID> /F
```

### Flutter can't connect to API

1. Check `app/.env` has correct `API_BASE_URL`
2. For emulator, use `http://10.0.2.2:3000`
3. Check API is running: open `http://localhost:3000` in browser
4. Check firewall isn't blocking port 3000

### Emulator not detected

```bash
cd app
fvm flutter doctor
fvm flutter devices
```

### Hot reload not working

- For API: nodemon should auto-restart on file changes
- For Flutter: Press `r` in terminal or save files in VS Code

### Database connection errors

1. Verify MySQL is running
2. Check credentials in `.env`
3. Ensure database exists
4. Check `API/src/db.js` for connection config
